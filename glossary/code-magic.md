# Code — 帶有力量的語言

## 定義
Code 是用來與機器溝通的精確語言。在本作的世界觀中，適當的 Code 可以改變現實，因此 **Code 是魔法的基礎**。

## 為什麼 Code 是魔法？

### 歷史脈絡：語言即力量

人類很早就相信精確的符號操作可以改變現實：

| 傳統 | 概念 | 與 Code 的對應 |
|------|------|----------------|
| **埃及象形文字** | 稱為「神的話語」(mdw nṯr)，文字寫下即具有力量 | 程式碼寫下即可執行 |
| **北歐盧恩文字** | 奧丁自掛世界之樹九天以獲取盧恩知識；每個字符蘊含宇宙力量 | 學習 coding 的代價與收獲 |
| **卡巴拉** | 上帝用 22 個希伯來字母創造世界；Golem 的額頭寫上「真理」即獲生命 | 程式碼作為「創世語言」 |
| **道教符籙** | 符咒須特定筆順、特定墨水書寫才生效 | 語法的精確性 |

### 核心法則

> *"Any sufficiently advanced technology is indistinguishable from magic."*
> — Arthur C. Clarke

反過來說：**任何足夠精確的語言，都具有改變現實的力量。**

### Code 與咒語的同構

| 魔法概念 | Code 對應 |
|----------|-----------|
| 咒語必須一字不差 | 語法錯誤 → 編譯失敗 |
| 知道事物的「真名」即可控制 | 正確的 API 呼叫 |
| 魔法書 / 魔典 (grimoire) | 程式庫 / 文件 (library / docs) |
| 學徒制 | 初心者 → 進階者的成長路徑 |
| 施法需要專注與意志 | Deep work、心流狀態 |
| 召喚 (conjure) | 動態生成物件 |
| 型別轉換 (casting) | Type casting — 「cast」同時是「施法」 |
| 十六進位 (hex) | 「hex」同時是「咒語」 |
| 守護程序 (daemon) | 背景中持續運行的「精靈」 |

## 在故事中的層次

### Level 0：基礎咒語
- 簡單的 script、Hello World
- Peter 目前在此階段
- 效果：能讓機器執行指令，偶爾產生預期外的效果

### Level 1：結構魔法
- 函式、物件、模組化
- 能組合基礎咒語成為更大的「儀式」

### Level 2：系統魔法
- 架構設計、API 串接
- 能讓不同的系統互相溝通——如同跨領域的魔法

### Level 3：生成魔法
- Prompt engineering、與 AI 對話
- 用語言引導另一個智能體——這不是控制，而是協作

### Level ???：夢境 Code
- 在夢境層中書寫的 code
- 效果未知——可能改變夢境本身的規則
- Peter 尚未觸及
