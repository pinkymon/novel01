# Session 02 技術名詞教學：Tool Use

> 參考：`reference/series1idea/s02-tool-use.md`

## 核心概念

**Tool Use** 是 Agent 從「只會說話」進化到「能做事」的關鍵一步。模型透過 Tool Dispatch 查找對應的工具函式，執行操作後將結果回傳。四個基礎工具——bash、read、write、edit——涵蓋了與外部世界互動的核心能力。

安全性透過 Path Sandboxing 確保，所有檔案操作都被限制在工作目錄內。

## 名詞表

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Tool Dispatch | N | 將工具名稱映射到執行函式的對應表 | 像是工具腰帶——每個口袋對應一種工具（暫定） |
| Handler Function | N | 與特定工具綁定的執行函式 | 每把工具的使用方式（暫定） |
| Path Sandboxing | N | 驗證檔案路徑不超出工作目錄邊界的安全機制 | 冒險者公會的結界（暫定） |
| Tool Schema | N | 定義工具輸入結構和參數規範的描述 | 工具的使用說明書（暫定） |
| Workspace Boundary | N | 所有操作必須限制在此邊界內的工作目錄 | 任務區域的邊界（暫定） |

## RPG 世界的理解方式

想像一個冒險者剛拿到工具腰帶。腰帶上有四個口袋，每個口袋裝著不同的工具。當冒險者說「我要讀這張地圖」，腰帶會自動找到對應的口袋，把工具遞出來。

但工具有使用規則——你不能用它去打開不屬於你的區域。結界會擋住你。這不是限制，是保護。

Loop 還是那個 Loop，但現在 Agent 多了一個能力：不只是說話，還能透過工具**作用於世界**。
