# Session 07 技術名詞教學：Task System

> 參考：`reference/series1idea/s07-task-system.md`

## 核心概念

**Task System** 將任務從記憶體搬到磁盤——存為 JSON 檔案的有向無環圖（DAG）。每個任務有狀態、依賴關係（blockedBy）、所有者。完成一個任務時，自動解除它阻擋的其他任務。

與 Session 03 的 TodoWrite 不同：TodoWrite 是記在腦中的便條紙（記憶體內，重啟就沒了），Task System 是刻在石板上的任務（磁盤上，永久保存）。

## 名詞表

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Task Graph | N | 以 JSON 檔案表示的有向無環圖 | 像是任務石板（暫定） |
| Dependency Resolution | N | 完成任務時自動解除其他任務的阻擋 | 解鎖——打倒守門怪後通道就開了（暫定） |
| blockedBy Edge | N | 任務間的有向依賴邊 | 任務之間的鎖鏈（暫定） |
| Task Persistence | N | 任務資訊存於磁盤，可存活於重啟 | 刻在石板上——失去意識醒來還在（暫定） |
| Parallel Execution | Adj | 無依賴的多個任務可同時運行 | 多線出擊（暫定） |

## RPG 世界的理解方式

之前冒險者把任務寫在筆記本上。問題是——如果你昏過去了（上下文壓縮），筆記本上的字也跟著消失。

任務石板不一樣。刻在石頭上的東西不會消失。而且石板之間有鎖鏈——「先打倒門口的守衛」這塊石板完成之前，「進入寶物庫」那塊石板是鎖住的。完成前者，後者自動解鎖。

這是從「記住要做什麼」到「系統性管理要做什麼」的跳躍。
