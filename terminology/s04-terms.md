# Session 04 技術名詞教學：Subagents

> 參考：`reference/series1idea/s04-subagents.md`

## 核心概念

**Subagent** 是父代理生成的獨立子代理，擁有自己的空白 messages[] 和工具集。關鍵設計：子代理完成後只回傳摘要（Summary Return），而不是把所有工具調用歷史塞回父代理的上下文。

這解決了「一個人做太多事會爆腦」的問題。同時，子代理不能再生成子代理（Recursive Spawning Prevention），防止無限嵌套。

## 名詞表

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Subagent | N | 擁有獨立訊息列表和工具集的子代理實例 | 像是分身術（暫定） |
| Context Isolation | N | 子代理的工具歷史不累積到父代理的訊息列表 | 分身的記憶不會回流（暫定） |
| Summary Return | N | 子代理將執行過程濃縮為單一段落回傳 | 分身回報（暫定） |
| Fresh Context | N | 子代理啟動時訊息列表為空 | 分身從零開始（暫定） |
| Recursive Spawning Prevention | N | 子代理不能再生成子代理的限制 | 分身不能再分身（暫定） |

## RPG 世界的理解方式

當任務太複雜，一個人做不完，冒險者學會了分身術——召喚出一個分身去偵查前方，自己留在原地。分身回來只說一句：「前面有三隻史萊姆，左邊有條通道。」

分身不會把偵查過程中的每一步都灌進你腦袋。這很重要——如果灌進來的話，你自己的記憶就爆了。

而且分身不能再分身，否則整個地下城會充滿你的複製品。
