# 技術名詞總表

> 按 Session 排列。每個技術名詞標注詞性、技術定義、以及在 Dragon Quest 故事中的類比。
> 詞性標記：名詞(N)、動詞(V)、形容詞(Adj)
> 類比欄位為柔性映射，標註「（暫定）」者待 Director 討論確認。

---

## Session 01 — The Agent Loop

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Agent Loop | N | 觀察→判斷→行動→回饋的無限循環，直到模型停止呼叫工具 | The Loop——世界運轉的法則，進入 Loop 的存在是「活的」 |
| messages[] | N | 累積式對話紀錄陣列，儲存所有輸入與輸出 | 像是冒險者的記憶——經歷過的事都留在腦海中 |
| stop_reason | N | 模型回傳的終止條件，決定 Loop 是否繼續 | Loop 的脈搏——當行動的理由消失，循環暫停 |
| Tool Call | N | 模型主動請求執行特定工具的操作 | 行動的意志——決定「做點什麼」而非「說完收工」 |
| Tool Result | N | 工具執行後返回的輸出，重新傳回模型作為下一輪上下文 | 行動的回饋——世界對冒險者動作的回應 |
| Exit Condition | N | 控制循環終止的判定邏輯 | Loop 的終點——什麼時候可以停下來 |

## Session 02 — Tool Use

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Tool Dispatch | N | 將工具名稱映射到執行函式的對應表 | 像是工具腰帶——每個口袋對應一種工具（暫定） |
| Handler Function | N | 與特定工具綁定的執行函式，接收輸入並返回結果 | 每把工具的使用方式——知道怎麼揮劍和怎麼讀地圖是不同的技能（暫定） |
| Path Sandboxing | N | 驗證檔案路徑不超出工作目錄邊界的安全機制 | 冒險者公會的結界——超出範圍的行動會被阻止（暫定） |
| Tool Schema | N | 定義工具輸入結構和參數規範的描述 | 工具的使用說明書——不照著用就會失敗（暫定） |
| Workspace Boundary | N | 所有操作必須限制在此邊界內的工作目錄 | 任務區域的邊界——出了這個範圍就不歸你管（暫定） |

## Session 03 — TodoWrite

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| TodoManager | N | 儲存任務列表並強制同時只有一個任務進行中的狀態管理物件 | 像是冒險者筆記本——寫下要做的事，一次只做一件（暫定） |
| Task Status | N | 任務的狀態標籤：pending、in_progress、completed | 任務的進度標記——待辦、進行中、完成（暫定） |
| Nag Reminder | N | 模型多輪未更新任務時系統自動注入的提醒 | 像是公會櫃台的催促——「你的任務進度呢？」（暫定） |
| Sequential Focus | N | 同時只有一個任務進行中的約束，避免上下文漂移 | 專注的力量——分心就會迷路（暫定） |
| Context Fading | N | 長對話中系統提示優先級逐漸降低的現象 | 記憶的模糊——走太遠就忘了出發的目的（暫定） |

## Session 04 — Subagents

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Subagent | N | 擁有獨立訊息列表和工具集的子代理實例 | 像是分身術——召喚出來執行特定任務的分身（暫定） |
| Context Isolation | N | 子代理的工具歷史不累積到父代理的訊息列表 | 分身的記憶不會回流——只有結果會傳回本體（暫定） |
| Summary Return | N | 子代理將整個執行過程濃縮為單一段落回傳 | 分身回報——「任務完成，結果如下」（暫定） |
| Fresh Context | N | 子代理啟動時訊息列表為空，不繼承父代理歷史 | 分身是從零開始的——沒有本體的記憶包袱（暫定） |
| Recursive Spawning Prevention | N | 限制子代理不能再生成子代理的設計 | 分身不能再分身——防止無限套娃（暫定） |

## Session 05 — Skill Loading

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Skill | N | 特定領域的知識單元，按需注入而非全部預載 | 像是圖書館的藏書——需要時才翻開（暫定） |
| On-demand Loading | Adj | 模型請求時才提供知識內容的載入方式 | 用到才學——不是把所有魔法書都背在身上（暫定） |
| Two-layer Injection | N | 第一層系統提示放名稱，第二層工具結果放完整內容 | 先知道有這本書，翻開才看到內容（暫定） |
| Frontmatter | N | YAML 格式的元數據區塊，定義技能屬性 | 書的封面和目錄——快速知道這本書講什麼（暫定） |
| Skill Metadata | N | 技能的組織信息，與技能本體分離以節省空間 | 圖書館的索引卡——不用翻開書就知道在哪（暫定） |

## Session 06 — Context Compact

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Context Compression | N | 管理有限上下文視窗的三層次壓縮策略 | 像是整理背包——空間有限，要學會取捨（暫定） |
| Micro-compact | N | 靜默將舊工具結果替換為佔位符的壓縮層 | 自動遺忘細節——只記得「做過」但忘了過程（暫定） |
| Auto-compact | N | 令牌數超過閾值時自動觸發的 LLM 總結壓縮 | 記憶的自動整理——太多了就壓縮成摘要（暫定） |
| Transcript Persistence | N | 完整對話記錄儲存於磁盤，壓縮後仍可恢復 | 日記本——就算記憶模糊了，翻日記還能想起來（暫定） |
| Token Threshold | N | 觸發自動壓縮的令牌數上限 | 背包的容量上限——超過就裝不下了（暫定） |

## Session 07 — Task System

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Task Graph | N | 以 JSON 檔案表示的有向無環圖，含狀態與依賴 | 像是任務石板——刻在石頭上就不會消失（暫定） |
| Dependency Resolution | N | 完成任務時自動解除其他任務的阻擋關係 | 解鎖——打倒守門怪之後通道就開了（暫定） |
| blockedBy Edge | N | 任務間的有向依賴邊，表示先後順序 | 任務之間的鎖鏈——A 沒做完就不能開始 B（暫定） |
| Task Persistence | N | 任務資訊存於磁盤而非記憶體，可存活於重啟 | 刻在石板上的任務——就算失去意識醒來還在（暫定） |
| Parallel Execution | Adj | 無依賴的多個任務可同時運行 | 多線出擊——沒有先後關係的事可以同時做（暫定） |

## Session 08 — Background Tasks

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Background Execution | N | 在後台線程中運行長時間操作，主線程繼續工作 | 像是派出信鴿——送出去之後不用站著等回信（暫定） |
| Daemon Thread | N | 不阻擋主執行迴圈的後台線程 | 影子分身——在背後默默工作的存在（暫定） |
| Notification Queue | N | 線程安全的佇列，存儲後台完成結果 | 信箱——信鴿回來的時候把結果放進去（暫定） |
| Blocking vs Non-blocking | Adj | 阻擋式等待操作完成 vs 並行運行異步注入結果 | 站著等 vs 邊走邊等——兩種不同的做事方式（暫定） |
| Task Spawning | V | 啟動後台任務並立即返回的操作 | 放出信鴿——發出指令後立刻去做下一件事（暫定） |

## Session 09 — Agent Teams

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Persistent Agent | N | 持續存活於應用生命週期的智能體，具身份識別 | 像是固定隊友——有名字、有專長、不會消失（暫定） |
| Team Mailbox | N | 基於檔案的非同步通訊機制 | 隊伍信箱——留言給隊友，他看到就會處理（暫定） |
| Inbox Draining | V | 讀取並清空訊息檔案，確保每條訊息只處理一次 | 收信——把信箱裡的信全部拿出來讀完（暫定） |
| Agent Lifecycle | N | 智能體的狀態機：生成→工作→閒置→關閉 | 隊友的一天——起床、工作、休息、就寢（暫定） |
| Message Bus | N | 協調多智能體通訊的中樞系統 | 公會的佈告欄——所有人都在這裡交換訊息（暫定） |

## Session 10 — Team Protocols

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Request-Response Protocol | N | 以唯一識別碼追蹤的非同步請求-回應協議 | 像是正式的委託書——有編號、有回覆、可追蹤（暫定） |
| FSM (Finite State Machine) | N | 管理請求狀態轉換的有限狀態機 | 委託的流程——提出→審核→批准或駁回（暫定） |
| Plan Approval | N | 提交執行計畫由領導者審查批准的機制 | 作戰計畫需要隊長批准才能執行（暫定） |
| Shutdown Handshake | N | 領導者發起、智能體確認的優雅關閉協議 | 撤退的暗號——確認所有人安全才能撤（暫定） |
| Request Correlation | N | 用唯一 ID 將請求與回應綁定的追蹤機制 | 委託編號——確保回覆對得上原本的請求（暫定） |

## Session 11 — Autonomous Agents

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Autonomy | N | 智能體自行掃描並認領任務，毋需逐一指派 | 像是自主行動的冒險者——不等命令，自己找活幹（暫定） |
| Idle Phase | N | 完成工作後定期輪詢尋找新任務的等待狀態 | 待命狀態——在公會裡等待新任務上榜（暫定） |
| Task Board Scanning | V | 定期搜索工作板中未認領任務的操作 | 看佈告欄——掃一眼有沒有適合自己的任務（暫定） |
| Identity Re-injection | N | 上下文壓縮後重新插入身份區塊以保持角色連貫 | 記住自己是誰——就算記憶模糊也不忘本（暫定） |
| Task Claiming | V | 透過更新所有者欄位來認領任務的動作 | 揭榜——在佈告欄的任務上寫下自己的名字（暫定） |
| Idle Timeout | N | 閒置狀態的最長等待時間，逾期自動關閉 | 等太久就回家——不能無限期守在公會裡（暫定） |

## Session 12 — Worktree + Task Isolation

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Worktree | N | Git 的隔離工作目錄，允許同一倉庫不同分支同時操作 | 像是平行維度——同一個世界的不同副本（暫定） |
| Task-Worktree Binding | N | 將任務與專屬隔離執行目錄綁定 | 每個任務有自己的維度——互不干擾（暫定） |
| Directory Isolation | N | 為每個任務分配獨立環境，防止並行衝突 | 維度隔離——在自己的空間裡怎麼改都不影響別人（暫定） |
| Execution Plane | N | 由隔離工作目錄組成的執行層，與控制平面分離 | 執行的世界——和計畫的世界是分開的兩層（暫定） |
| Event Stream | N | 記錄工作樹生命週期事件的日誌 | 維度日誌——記錄每個平行世界發生了什麼（暫定） |
| Worktree Registry | N | 記錄所有活躍工作樹元數據的中央索引 | 維度管理局——知道目前有哪些平行世界在運作（暫定） |
