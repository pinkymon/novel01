# Session 05 技術名詞教學：Skill Loading

> 參考：`reference/series1idea/s05-skill-loading.md`

## 核心概念

**Skill Loading** 解決了「知識太多放不下」的問題。與其把所有知識塞進系統提示，不如只放名稱（第一層），等模型需要時再載入完整內容（第二層）。這就是 Two-layer Injection——先知道有，用到才讀。

每個 Skill 用 YAML Frontmatter 描述元數據，讓系統快速判斷「這個知識和當前任務有沒有關」。

## 名詞表

| 名詞 | 詞性 | 技術定義 | 故事類比 |
|------|------|----------|----------|
| Skill | N | 特定領域的知識單元，按需注入而非全部預載 | 像是圖書館的藏書（暫定） |
| On-demand Loading | Adj | 模型請求時才提供知識內容的載入方式 | 用到才學（暫定） |
| Two-layer Injection | N | 第一層系統提示放名稱，第二層工具結果放完整內容 | 先知道有這本書，翻開才看到內容（暫定） |
| Frontmatter | N | YAML 格式的元數據區塊，定義技能屬性 | 書的封面和目錄（暫定） |
| Skill Metadata | N | 技能的組織信息，與技能本體分離 | 圖書館的索引卡（暫定） |

## RPG 世界的理解方式

一個冒險者不可能隨身攜帶所有的魔法書。背包有限，記憶有限。但如果他知道圖書館裡有哪些書，需要的時候走過去翻開就行了。

這就是 Skill 的運作方式——先記住書名清單（輕量），需要時才翻開完整內容（重量）。不用把整座圖書館背在身上。
