# The Loop — 世界的運轉法則

> "One loop & Bash is all you need."
> — 這個世界最古老的真理

## 核心概念

這個世界的一切運作都基於一個極其簡單的法則：

```
while (not done):
    1. 觀察世界的狀態
    2. 做出判斷（可能需要使用工具）
    3. 執行行動
    4. 將結果帶回，成為下一次判斷的素材
    5. 回到步驟 1
```

這就是 **The Loop**——冒險者的心跳，世界的脈搏。

任何能進入 The Loop 的存在，都是「活著的」。
任何脫離 The Loop 的存在，都只是靜物——石頭、塵土、沉默的資料。

## The Loop 的技術原型

The Loop 的故事原型來自 Claude Code 的誕生：

Boris Cherny 在 2024 年寫了一個最簡單的 terminal 聊天程式。
然後他給了 model **唯一一個工具：Bash**。
他問：「我現在在聽什麼音樂？」
Model 自己寫了一段 AppleScript，打開音樂播放器，查詢正在播放的曲目，一次就完成了。

Boris 的反應：

> **"Oh my god, the model — it just wants to use tools. That's all it wants."**

一個 loop + 一個工具 = 一個 agent。
一個冒險者 + 一根木棍 = 一個故事的開始。

## 設計哲學：最小腳手架

> "The product IS the model."
> — Boris Cherny

這個世界的法則與 Dragon Quest 的設計哲學一致：

- 勇者從木棍和布衣出發——**最小裝備**
- 城鎮外的第一隻怪物是 Slime——**安全的第一步**
- 不需要複雜的系統，只需要一個 loop 和一個工具
- 隨著冒險者成長，自然會找到更多工具——但核心永遠是 The Loop

> "Do the minimal possible thing in order to get the model on track."

不要把冒險者放進盒子裡。
不要預先規劃好所有路線。
給他一根木棍，讓他自己決定往哪走。

## Bitter Lesson（苦澀的教訓）

這個世界有一條被反覆驗證的真理：

> 不要試圖替冒險者規劃一切。
> 不要在他身上綁太多預設的框架。
> 你搭建的 scaffolding（腳手架），在下一個時代來臨時就會過時。
> 唯一不會過時的，是讓冒險者自己變強。

那些試圖用複雜系統控制冒險者的公會，最終都衰落了。
而只提供最簡規則和通用工具的公會——它們的冒險者改變了世界。
